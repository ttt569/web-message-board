# web-message-board
基于纯前端技术（HTML + CSS + JavaScript）实现的留言板系统，所有数据均存储在浏览器 `localStorage` 中，无需后端服务器。系统包含用户注册、登录、留言发布、点赞、评论等完整功能。
# 留言板系统 — 项目说明文档

## 一、项目概述

本项目是一个基于纯前端技术（HTML + CSS + JavaScript）实现的留言板系统，所有数据均存储在浏览器 `localStorage` 中，无需后端服务器。系统包含用户注册、登录、留言发布、点赞、评论等完整功能。

## 三、功能模块详解

### 3.1 用户注册（Register）

| 项目 | 说明 |
|------|------|
| 页面 | `Register.html` |
| 脚本 | `Register.js` |
| 样式 | `Reg-Log.css` |

**校验规则：**

| 字段 | 校验条件 | 错误提示 |
|------|----------|----------|
| 手机号 | 非空 + 正则 `/^1[3-9]\d{9}$/` | "请输入手机号" / "手机号格式错误" |
| 用户名 | 非空 + 正则 `/^[a-zA-Z0-9_\u4e00-\u9fa5]{3,16}$/` | "请输入用户名" / "用户名至少3位,支持字母、数字、下划线或中文" |
| 密码 | 非空 + 正则 `/^(?=.*[a-zA-Z])(?=.*\d).{6}$/` | "请输入密码" / "密码至少6位,必须包含字母和数字" |

**注册流程：**
1. 填写手机号、用户名、密码
2. 依次校验手机号 → 用户名 → 密码，任一不通过则终止
3. 检查用户名是否已存在
4. 检查手机号是否已被注册
5. 构建用户对象，存入 `localStorage`
6. 注册成功后延迟 1 秒跳转到登录页

**用户数据结构：**
```json
{
    "username": "string",
    "password": "string",
    "phone": "string",
    "registerTime": "ISO时间字符串"
}
```

### 3.2 用户登录（Login）

| 项目 | 说明 |
|------|------|
| 页面 | `Login.html` |
| 脚本 | `Login.js` |
| 样式 | `Reg-Log.css` |

**校验规则：**

| 字段 | 校验条件 | 错误提示 |
|------|----------|----------|
| 手机号 | 非空 + 正则 `/^1[3-9]\d{9}$/` | "请输入手机号" / "手机号错误" |
| 密码 | 非空 | "请输入密码" |

**登录流程：**
1. 填写手机号、密码
2. 格式校验通过后，从 `localStorage` 中查找匹配用户
3. 手机号未注册 → 提示"该手机号未注册"
4. 密码不匹配 → 提示"密码错误"
5. 登录成功 → 将 `isLoggedIn`、`username` 写入 `localStorage`
6. 延迟 500ms 跳转到留言板页面

**实时校验：** 手机号和密码输入框在失焦（`blur`）时自动触发校验。

### 3.3 留言板（Message）

| 项目 | 说明 |
|------|------|
| 页面 | `Message.html` |
| 脚本 | `Message.js` |
| 样式 | `Message.css` |

#### 3.3.1 功能清单

| 功能 | 说明 | 权限 |
|------|------|------|
| 发布留言 | 输入内容后点击提交，新留言置顶显示 | 需登录 |
| 删除留言 | 仅留言作者可删除自己的留言 | 留言作者 |
| 点赞/取消点赞 | 切换点赞状态，显示点赞人数 | 需登录 |
| 评论 | 在留言下方发表评论 | 需登录 |
| 登录/退出 | 顶部显示登录状态，可切换登录/退出 | 所有人 |

#### 3.3.2 留言数据结构

```json
{
    "id": 1718400000000,
    "username": "张三",
    "content": "这是一条留言",
    "time": "2025/6/15 10:00:00",
    "likes": ["用户A", "用户B"],
    "comments": [
        {
            "id": 1718400010000,
            "username": "李四",
            "content": "这是一条评论",
            "time": "2025/6/15 10:01:00"
        }
    ]
}
```

#### 3.3.3 核心设计模式

**1. 模板克隆渲染**

使用 HTML `<template>` 标签定义留言卡片和评论项的模板，通过 `cloneNode(true)` 克隆模板后填充数据，避免字符串拼接带来的 XSS 风险。

**2. 事件委托**

留言列表中的所有点击操作通过 `data-action` 属性标识，统一在 `messageList` 容器上监听 `click` 事件，根据 `action` 值分发到对应处理函数：

| data-action | 功能 |
|-------------|------|
| `delete` | 删除留言 |
| `like` | 点赞/取消点赞 |
| `toggle-comment` | 展开/收起评论区 |
| `add-comment` | 提交评论 |

**3. 键盘快捷提交**

评论输入框支持 `Enter` 键提交，通过 `data-action="comment-keydown"` 标识，在 `keydown` 事件中统一处理。

## 四、localStorage 数据说明

| Key | 类型 | 说明 |
|-----|------|------|
| `users` | `JSON Array` | 所有注册用户列表 |
| `messages` | `JSON Array` | 所有留言数据（含评论） |
| `isLoggedIn` | `string` | 登录状态，值为 `"true"` 或不存在 |
| `username` | `string` | 当前登录用户的用户名 |

## 五、页面导航关系

```
注册页面 (Register.html)
    │ 注册成功后跳转
    ▼
登录页面 (Login.html)
    │ 登录成功后跳转

```

## 六、技术要点总结

| 要点 | 实现方式 |
|------|----------|
| 数据持久化 | `localStorage` + `JSON.stringify/parse` |
| XSS 防护 | `escapeHtml()` 函数 + `textContent` 赋值（而非 `innerHTML`） |
| DOM 渲染 | `<template>` 模板克隆 + 数据填充 |
| 事件处理 | 事件委托模式，减少事件监听器数量 |
| 唯一 ID | `Date.now()` 时间戳 |
| 表单校验 | 正则表达式 + 逐级校验 + 失焦实时校验 |
| 状态管理 | `localStorage` 存储登录状态，页面加载时读取并更新 UI |
| 评论区状态保持 | 提交评论后通过 `setTimeout` 重新展开评论区 |
